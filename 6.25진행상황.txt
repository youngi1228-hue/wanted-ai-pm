[데이터 수집 및 솔루션 활용 계획]
1. 데이터 출처
교육청은 관련 직업이 없어 Wanted로 선정

2. 활용 솔루션
Python, Streamlit/NiceGUI

3. 선정 이유 
배웠던 파이썬을 활용하여 연동이 잘 된다는 Streamlit을 사용해 봄.

4. 레이아웃

이력서 입력
        ↓
Wanted 검색 결과 페이지
        ↓
Firecrawl scrape_url()
        ↓
채용공고 URL 추출
        ↓
각 공고 상세페이지 재수집
        ↓
입력된 이력서 내용과 매칭하여 연관 점수가 높은 순으로 맞춤 공고 추천 페이지 만듦. URL을 클릭해서 볼 수 있음.

5. 데이터 분석 
Wanted의 AI PM 관련 채용공고를 대상으로 데이터를 수집

6. 정제 기준
제목, 회사명, 지역, 경력, 주요업무, 자격요건, 우대사항, 공고 URL 등
불필요한 줄바꿈, 중복 공백, 마크다운 기호, 이스케이프 문자 등 화면 표시와 분석을 방해하는 문자를 제거
주요 기술, 직무, 산업 관련 단어를 키워드로 정리하여 검색과 이력서 매칭에 활용할 수 있도록 함
매칭 점수와 함께 어떤 키워드가 일치했는지를 표시하여 추천 결과의 근거를 확인할 수 있도록 함

7. 문제점 및 느낀점
직접적으로 크롤링을 하면 차단될 수 있다고 하여 우회하는 방법을 사용해보고, UI까지 구현해 볼 수 있어 매우 흥미로웠음.
툴을 다양하게 사용해보기 위해 Streamlit 대신 Flask 또는 FastAPI보다 작업량이 적은 NiceGUI로 툴을 변경해서 구성해 보았는데 Streamlit에 비해 디자인의 질이 떨어진다는 것을 알게 됨.

8. 향후 방향
Streamlit Community Cloud와 Activepieces Cloud를 이용해서 만들어 보려고 함.

**수정사항**
이력서 작성
→ 수신 이메일 저장
→ 매칭 실행
→ 추천 결과 TOP 5 생성
→ 이메일로 받기 버튼
→ Gmail SMTP 발송



7. 코드 전달 없이 같은 장소에서 시연하는 방법

팀원들이 같은 와이파이나 사내 네트워크에 연결되어 있다면 본인 컴퓨터에서 앱을 실행하고 접속 링크만 전달할 수도 있습니다.

.\.venv\Scripts\python.exe -m streamlit run dashboard.py `
    --server.address 0.0.0.0 `
    --server.port 8501

본인 컴퓨터의 IP 주소를 확인합니다.

ipconfig

IPv4 주소가 예를 들어 192.168.0.15라면 팀원은 브라우저에서 다음 주소로 접속합니다.

http://192.168.0.15:8501

이 방식은 다음 조건에서만 작동합니다.

같은 네트워크에 연결되어 있음
본인 컴퓨터에서 Streamlit이 실행 중임
Windows 방화벽에서 Python 또는 8501 포트 접근을 허용함
사내 네트워크가 기기 간 접속을 차단하지 않음

원격 접속 시에는 해당 포트가 네트워크와 방화벽에 열려 있어야 합니다.

8. 팀원들이 인터넷 링크로 접속하게 하려면

나중에는 GitHub 저장소를 Streamlit Community Cloud에 연결해 웹 링크로 배포할 수 있습니다. 배포에는 저장소 구조와 requirements.txt가 필요하고, .env 대신 Cloud의 Secrets 설정 화면에 SMTP 값을 입력해야 합니다.

Cloud Secrets에는 다음처럼 등록할 수 있습니다.

SMTP_SENDER_EMAIL = "발송계정@gmail.com"
SMTP_APP_PASSWORD = "구글앱비밀번호"
SMTP_SENDER_NAME = "커리어 에이전트"

Streamlit의 최상위 Secrets 값은 환경변수로도 제공되므로 현재 os.getenv() 방식과 연결할 수 있습니다.

다만 현재는 코드가 계속 수정 중이므로, 먼저 비공개 GitHub에서 팀원들이 로컬 실행 → 기능 확정 후 온라인 배포 순서가 가장 안전합니다.
